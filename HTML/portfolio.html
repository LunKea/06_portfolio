The goal for the first project

For the first project, as an introduction to HTML and CSS, we had to make a website using those type of files, following wireframes and layout-diagrams that we were handed by the teachers.

We were also told to make the website mobile-first and make it responsive so i can be on mobile and desktop.

Layout-Diagrams

In this course, we were introduced to layout-diagrams, which was my first time seeing but i could easily understand how helpful it is when it comes to coding and found it really handy to have. I learned how to implement this for this project and also how to apply it to future websites/projects.

We all had to use the same class, id and layout-diagram. Which helped us in working together in case anyone ran into a problem.

Wireframes

Next to layout-diagrams we learned about wireframes, a concept I’m already familiar with as I’ve made apps before, so I still had something to learn as it was different from when I used it to make apps.

The wireframe showcases how the product will look like with headers, paragraphs and images. This wireframe we were handed was heavily focused on grids, as we were also introduced to CSS grids in this project.

As practice for CSS grids, we did a lot of CSS grid exercises with hints on how to solve the exercises. In my opinion the exercises helped a lot and prepared me well.

Mobile-first

I also learned about doing mobile-first when it comes to making a responsive website, as it helps out with the process of making a website, since the mobile-version is most likely going to be the simpler version of the website, plus making sure that a burger menu can be implemented later.

Doing mobile-first also just helps with transitioning from mobile to desktop, as the mobile-setup can sometime easily transfer to desktop if the website is responsive.

Responsiveness and mediaquery

In mobile-first i mentioned making the website responsive, but i didn’t mention how you do that. Through the CSS file you write a mediaquery, which will make a rule dependent on the width or height, but we used width. So dependent on the width of the browser, the layout will change.

In the future it can later be used for: Desktop into tablet into phone.

Media query also helps with changing styles such as font-size and image size depending on what device you view the website on.

Making galleries with CSS and grids

With the CSS grid exercises we did, we ended up using them for the gallery of the website, with span and placement within a grid.

It was pretty simple to understand, especially because the exercises made me familiar with the concept of how to place and span images across the grids.

I noticed some of my classmates struggled with it when it came to making it, they had to end up drawing up the span but I could undestand it to the point of knowing how to make it from the get go, though I do have to admit I did run into some problems when it came to spanning on height.

Reflection on the project

As I was completely new to this I did struggle a lot with understanding how CSS grids worked in the beginning but as I kept struggling in the end I learned how to utilize CSS grids.

My biggest challenge during this course was actually learning to set up my HTML as I originally made my HTML really weird and complex to the point where no one could help me, so I had to re-do my HTML. But doing so I learned how to use the section and div tag properly and how to make grids!

One other challenge I also had throughout this course was using the flex property.

Ohter than setting up HTML and CSS grids, we learned a lot of CSS properties which I had a lot of fun with. We learned how to make hover effects, how to link websites through text, image properties in CSS, how to make a footer/header and my favorite part: how to insert fonts.

We also learned how to make our website responsive so it can work both on desktop and mobile.

Choosing a topic and the purpose

So the topic/theme of my website was for this Manga (Japanese Comic) that I am a fan of called: “The Guy She Was Interested In Wasn’t A Guy At all”

The reason why I chose it is because I thought it could be a really fun website to create, as I am heavily involved with the community around the product and it has a very strong and distinctive visual identity which is one of the reason why i’m a fan.

With that being said, I figured out the purpose of the website should be some sort of guide on how to read it online and how to get the physical version of the manga as well, since this manga just recently released physically when i made the website.

User-type, target audience & user stories

First thing I did after deciding what website to make, I went to look at other websites with the same purpose as mine and stumbled upon a guide on how to buy the manga-series: “LOVE BULLET”. Which is a very similar product to mine as both mangas have the audiences that align with each other, being the LGBTQ community.

Moodboard

Once I got my feedback, we then learned about making moodboards based on keywords you chose, in this project we were told to pick 3 key-words.

I really like this method as I find it really helpful to narrow down how you want your website/product to look like, especially with the key-words in mind.

We were told to pick 3 but I ended up using 2, as those 2 keywords was the most important to my product’s identity.

I ended up making the website based on the key-words: Comic and Music.

Style Tile

After getting feedback on the moodboard I would then create a style tile based on how the website would look, showcasing the:

  • The Buttons(CTAs)
  • Layout of the landing page


I also created the style tile based on a moodboard of the product itself, which included the colors, textures, accents and small details.

Finalizing style tile

I had two versions of my style tile, so to find out which version to go with I would create a 5-second test, also to see what the visitor’s first impression of the website was.

I found the 5-second test to be an amazing data-collection on first impression and what gets noticed first by people who visits the website, it could narrow down if i my intentions correctly or not. As my website had a strong color, I wanted to figure out if people felt like it was very harsh to look at on first impression, because if it was then it wouldn’t be a pleasing experience on a longer visit.

Hi-fi prototype

After all of my research I would then create a hi-fi prototype of the website, where I would apply all of my CSS styles and write the text that would be on the website too.

I then did think-aloud test on my target audience, reaching out to them on X.com and asking for people to participate where I would give them a shout out on my own account after. Vector art with Illustrator

During this project we started to use our Adobe Illustrator and learned about the pen tool to make vector art, that we would be using for our game through .SVG files so it wouldn’t get pixelated if we were to up/down-scale the image.

The useage of .SVG file was amazing as our illustrations was now vector based so nothing would be lost on the picture quality when we’d change the size of it.

State Machine Diagram

Since we were working on a game we had to learn how to use two diagrams.

One of them being State Machine Diagram.

For the project’s context, the Sate Machine Diagram was used to guide us on what would happen when you interacted with each element that would be in our game.

It was also used as a tool for us to get help from our teachers if we ran into any problems. The diagram was really useful to have an overview of each element in the game

Activity Diagram(Lo-fi)t

The second diagram we learned to use was the Activity Diagram.

Activity Diagram is quite similar to State Machine Diagram, but it was made more on a non-coded way. As in it wouldn’t have any coding on the diagram and only show wat would be happening as an user.
